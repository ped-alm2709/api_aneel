Sobre a Aplicação


1. REQUISIÇÃO E ORGANIZAÇÃO DOS DADOS

  • Use a API da ANEEL para requisição de dados do pacote “relação de empreendimentos de geração distribuída”.

  • Obs: Sugerimos a utilização do “Python” para o consumo de API, mas poderá ser usada a linguagem de programação em que achar mais adequada.

  • Links de referência:

    1. Dados abertos da ANEEL
    2. Documentação sobre o CKAN

  • Agregar os dados em base mensal

  • O Total de empreendimentos por classe (Residencial, comercial, “...”) e estado federativo.

  • A Potência instalada de cada estado

  • Obs: Uso das colunas “DthAtualizaCadastralEmpreend”, ”DscClasseConsumo“, “MdaPotenciaInstaladaKW” e “SigUF”.

  • Organização dos dados para estrutura similar a banco de dados SQL: exportar dados em arquivos “json” e “csv”.

2. EXPOSIÇÃO DOS DADOS EM FORMA GRÁFICA

  • Selecione 2 das opções abaixo:


    1. Elaboração de um gráfico mostrando a evolução temporal do Total Mensal de Empreendimentos por Estado e por Classe de Consumo;

    2. Elaboração de um gráfico mostrando a evolução temporal da Potência Instalada Total (KW) por Estado;

    3. Obrigatória: Selecione variáveis de forma livre e faça a exposição da maneira que achar mais adequada (Será adotado como diferencial o uso da criatividade).

  • Sugerimos o uso de html, CSS, JavaScript, etc, porém a exposição dos dados em forma gráfica poderá ser feita através de outras ferramentas gráficas.

  • Informações a serem enviadas:


    • Códigos elaborados de Back e Front-end

    • Obs: Os arquivos devem ser de fácil interação, a preferência para acessar o conteúdo em Front-end (como os gráficos) é que não tenha necessidade de instalar bibliotecas, softwares e afins (por exemplo: o Front-end elaborado em html deve abrir uma aba no navegador com o conteúdo produzido);

    • Dados exportados;

    • Link de outras plataformas de DashBoard, caso utilizados.
